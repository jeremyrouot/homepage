Le langage HTML

ISEN Brest - CIR1

Présentation

  • Hypertext Markup Language
  • Inventé en 1991 par Tim Berner-Lee
  • Permet de structurer sémantiquement et de mettre en forme le contenu des pages
  • Permet d’inclure des ressources multimédias, des formulaires de saisie et des programmes informatiques
  • Documents interopérables avec des équipements très variés

Normalisation

  • IETF : Internet Engineering Task Force
  • WHATWG : Web Hypertext Application Technology Working Group
  • W3C : World Wide Web Consortium http://www.w3.org

Historique

  • 1991 : Création par Tim Berner-Lee au Centre Européen de Recherche Nucléaire (CERN)
  • 1995 : HTML 2.0 Normalisation par l’IETF
  • Début 1997 : HTML 3.2 Ajout des tables, des applets, etc.
  • Fin 1997 : HTML 4.0 Ajout des feuilles de styles, des frames. Séparation de la présentation et du contenu.
  • Décembre 1999 : HTML 4.01
  • 2000 : XHTML 1.0 Reformulation de HTML 4 en XML
  • 2006 : XHTML 2.0
  • 2014 : HTML 5

Normalisation par le W3C depuis 1996

HTML 5

  • Faire évoluer HTML pour décrire la sémantique des documents mais aussi les applications en ligne
  • Simplification et priorité donnée aux besoins utilisateurs
  • Évolutions plutôt que modifications radicales
  • Nouveaux éléments et nouveaux attributs

Édition d’une page HTML

  • Éditeurs de texte : Notepad, Notepad++, vi, Atom, ...
  • Éditeurs en ligne : Mozilla Thimble, JSFiddle
  • Éditeurs WYSIWYG
  • Systèmes de publication de contenu : CMS, blog

Première page HTML


<!doctype html>
<html>
	<head>
	<meta charset="utf-8">
	<title>Ma première page WEB</title>
	<meta name="author" content="moi">
	</head>
	<body>
	<h1>Ma page</h1>
	<p>
		Bonjour le monde !
	</p>
	</body>
</html>

Transport d'une page HTML

URL

  • Souvent appelé « adresse web »
  • Désigne une chaîne de caractères utilisée pour adresser les ressources du World Wide Web

Composé de :

  • Protocole de communication : http, ftp, file, ldap, mailto, telnet, etc.
  • Nom d’utilisateur (optionnel)
  • Mot de passe (optionnel)
  • Adresse IP ou nom de domaine
  • Numéro de port (optionnel)
  • Chemin d’accès à la ressource

Caractères spéciaux : % et code ASCII ou UTF-8

Exemple

http://gaston:123456@www.lagaffe.fr:8888/chemin/
	d/access.php.?var1=val1"var2=val2#signet

    • Nom de domaine

URL relative

  • Ne contient pas le protocole et le nom de domaine
  • Déduits à partir de la ressource contenant l’URL absolue
  • Souvent utilisé à l’intérieur d’un même site Web (évite la réécriture des URL en cas de déplacement).
  • Inspiré du système de fichier Unix : 
    • ./
      dossier actuel
    • ../
      dossier parent
    • /
      dossier racine

Structure du langage

Une page HTML est un document texte constitué d’éléments eux-mêmes constitués de balises et d’attributs permettant d’apporter de l’information sur le contenu.

Séparation de la structure et de la présentation

  • La structure d’un document est différente de son apparence
  • Un paragraphe est une structure dont l’apparence est différente en français et en américain
  • HTML décrit la structure du document
    • Historique : initialement dérive du langage documentaire SGML
    • Portabilité : indépendant des navigateurs
    • Libre choix de l’utilisateur : document visuel ou sonore par exemple en cas de handicap

Séparation de la structure et de la présentation

  • Structure : HTML
  • Présentation : feuilles de style CSS

Recommandé par le W3C

Validation du code

  • Le W3C fournit un outil : http://validator.w3.org
  • Pour les normes autre que HTML 5, il existe une DTD : un document décrivant la grammaire du langage
  • Permet également de valider l’accessibilité d’un site

Accessibilité

  • La loi du 11 février 2005 pour l’égalité des droits et des chances, la participation et la citoyenneté des personnes handicapées
  • Loi entrée en application le 14 mai 2009. Elle impose aux sites des administrations et des collectivités territoriales d’être en conformité avant le 14 mai 2012
  • Importance de la structure du code : alternatives aux images, résumé pour un tableau, etc.

Structure et principes de base

Éléments

Partie structurée du document

  • Contient des attributs et du contenu
  • Représente diverses structures : en-têtes, paragraphes, liens hypertextes, listes, images, etc.

Éléments

Type d’éléments

  • Bloc (block) : grandes structures contenant d’autres blocs, éléments texte ou du texte (titre, paragraphes, etc.)
  • En ligne (inline) : petites structures contenant d’autres éléments texte ou du texte (hyperliens, citations, images, etc.)
  • Auto-fermant : pas de contenu, uniquement pour la structure (saut de ligne, tabulation, etc.)

Balises

Permettent de délimiter des éléments

Syntaxe

  • Balise ouvrante (en début) : 
    <balise>
  • Balise fermante (en début) : 
    </balise>
  • Exemple : 
    
<p> Un paragraphe </p>

Syntaxe

  • Balise seule : 
    <balise>
  • Exemple : 
    
<br>

Attributs

Caractéristiques de l’élément : référence vers une adresse web (lien hypertexte), emplacement d’un fichier (image), etc

Syntaxe :

<balise attribut1="valeur1" attribut2="valeur2" attribut3="valeur3">texte</balise>

Exemple :


<img src="isen.jpg" alt="Logo ISEN"/>

Les formulaires

  • Introduction de l'interactivité
  • Elément <form> dont les attributs sont :
    • action : URL chargée du traitement des données
    • method : méthode d’acheminement des données GET ou POST
  • GET (par défaut) : transmis dans la partie query de l’URL donc visible
  • POST : transmis au serveur dans le flot de données, invisible

Enctype : méthode d’encodage pour la forme que prendra le contenu du formulaire (uniquement pour POST)

  • application/x-www-form-urlencoded (par défaut) : contenu du formulaire sous forme URL (exemple : nom=valeur_nom&prenom=valeur_prenom)
  • Text/plain : contenu du formulaire sous forme de texte lisible (utilisé pour l’action mailto)
  • Multipart/form-data : permet d’expédier un fichier attaché au message
